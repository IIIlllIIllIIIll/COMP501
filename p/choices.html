Choices for developers

Developers have the most control over the software, thus their actions could greatly improve this situation and reduce risks. Some choices including:

  1. Develop and use advanced static code analysis (Wu & Lu, 2020)
  2. Increase test coverage (Wu & Lu, 2020)
  3. Accepting certain preventive patches (Wu & Lu, 2020)
  4. Raising developers' risk awareness (Wu & Lu, 2020)
  5. Standardize code merge procedure, and require more developers to review before the merge (Lins et al., 2024)

One way to mitigate the risks is to introduce advanced static code analysis. By using code analysis tools before every merge, vulnerabilities could be discovered in an early stage.

Tests are another powerful tool to detect vulnerabilities. Developers could use test tools, such as fuzzers, to detect whether the codes have unusual behaviours.

Sometimes, this kind of vulnerability is introduced gradually. Accepting preventive patches could reduce the risks.

It is also important to let developers understand this kind of attack so that they can be prepared for this kind of attack.

Finally, a standardized procedure enables better resistance to attacks.
